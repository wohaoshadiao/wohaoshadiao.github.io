安装hexo

  1. 安装hexo时候报错

可能情况

  • node环境没有配置好
  • 安装命令不对

解决办法：在命令的前面加上 **npx hexo init blog ** (blog 为你自己创建的文件夹)

  1. 进入blog，安装相关依赖
  2. 打开hexo server，会给我们一个访问端口

  1. 将hexo部署到github，使每个人都可以访问

  2. 提交到github上去

有可能会提示缺少相关的依赖，按照上面的命令重新安装即可

  1. 安装好依赖之后，输入以下命令即可进入登录界面

  2. 按照自己的 https://user.github.io登录即可

如何更换自己的主题

选择你喜欢的主题，这里我选择的是butterfly主题，官网是点这里

然后进入主题github，克隆所需要的主题库

然后我们在config文件中配置好主题，hexo g -d 生成静态文件发现打开网页出现 提示“extends includes/layout.pug block content include includes/recent-posts.pug include includes/pagination.pug”

  • 原因是缺少相关的依赖

npm install hexo-renderer-pug hexo-renderer-stylus –save

每次修改过主题都要执行 hexo clean hexo g –重新生成静态界面 hexo s —打开本地查看是否运行成功

  • 为了減少升级主题带来的不便 (选做)：把 theme 文件夹中的 _config.yml 复制到 Hexo 根目录下，同重命名为_config.butterfly.yml — 这里复制的是主题文件夹下的_config，不是根目录

  1. Hexo会自动合并上下两个文件之中的配置，如果存在同名的配置，则优先使用_config.butterfly.ymml文件，优先级高
